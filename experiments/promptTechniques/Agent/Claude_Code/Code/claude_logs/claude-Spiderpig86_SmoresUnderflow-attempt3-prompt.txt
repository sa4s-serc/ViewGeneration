## SmoresUnderflow: Project Overview Report

SmoresUnderflow is a StackOverflow clone built with a microservice architecture, aiming for scalability, fault tolerance, and efficient resource utilization. The core functionality includes user registration/authentication, question/answer posting, upvoting/downvoting, and search capabilities.

**Key Components and Functionalities:**

*   **User Management:** The `su-users` microservice handles user registration, authentication (using JWT tokens stored in cookies and local storage), and profile management. The `su-accounts` service manages user accounts.
*   **Question and Answer Handling:** The `su-questions` and `su-answers` services manage question and answer-related operations, serving data to the frontend. Asynchronous insertion of questions and answers into the database and ElasticSearch is handled by the `qu-questions` and `qu-answers` queue services, respectively, using RabbitMQ for message queuing. A `qu-delete-questions` queue also exists for handling question deletion.
*   **Search Functionality:** The `su-search` service leverages ElasticSearch to provide efficient indexing and searching of questions and answers.
*   **Media Handling:** The `su-media` service manages media uploads and storage.
*   **Frontend:** The `su-frontend` directory contains the React application, providing the user interface. Key components include:
    *   `src/index.tsx`: Entry point of the application.
    *   `src/App.tsx`: Main application component defining routes and handling authentication.
    *   `src/pages/`: Contains page-level components for Login, Register, Home, Ask, Questions, and Search.
    *   `src/duck/`: Redux ducks for managing application state.
    *   `src/components/`: Reusable React components.
    *   `src/utils/`: Utility files, including constants and authentication helpers. A service worker is implemented for offline functionality.

**Architectural Design and Technologies:**

*   **Microservices:** The application adopts a microservice architecture, dividing functionality into independently deployable services for better scalability and maintainability.
*   **Message Queuing (RabbitMQ):** RabbitMQ facilitates asynchronous communication between microservices, specifically for tasks like adding and deleting questions/answers, promoting scalability and fault tolerance.
*   **Caching (Redis):** Redis is employed to cache frequently accessed data (e.g., user information, questions) to improve performance and reduce database load.
*   **Search Indexing (ElasticSearch):** ElasticSearch enables efficient and scalable search capabilities for questions and answers.
*   **Load Balancing (Nginx):** Nginx acts as a load balancer, distributing traffic across multiple instances of the microservices to ensure high availability and performance.
*   **Frontend Technologies:** The frontend utilizes React for building the user interface, Redux for centralized state management, and React Router for client-side routing.

**Deployment and Configuration:**

*   **Deployment Automation:** Ansible and PM2 are used for automated deployment and management of the application, enabling easy scaling and updates.
*   **Configuration Management:** The `config/` directory contains configuration files for Ansible, Nginx, and MongoDB.
*   **Infrastructure Configuration:** The `deployment/` directory includes Ansible playbooks for instance setup and PM2 configuration.
*   **Nginx Configuration:** The `nginx/` directory holds Nginx configuration files for load balancing and reverse proxying.
*   **Database:** MongoDB is used as the primary database, with configuration and initialization scripts located in the `mongo/` directory.

**Key Design Patterns:**

*   **Microservices:** Dividing the application into independently deployable services.
*   **Message Queue:** Utilizing RabbitMQ to decouple services and handle asynchronous tasks.
*   **Caching:** Implementing Redis to improve performance by caching frequently accessed data.
*   **Load Balancing:** Employing Nginx to distribute traffic across multiple instances.
*   **Redux:** Centralized state management in the frontend application.
*   **JWT Authentication:** Securing the API using JSON Web Tokens.
