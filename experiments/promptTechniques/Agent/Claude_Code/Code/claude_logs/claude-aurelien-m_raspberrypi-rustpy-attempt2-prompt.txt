**Overall Project Summary**

This repository implements a Raspberry Pi hardware control system, primarily focused on LED manipulation, using a combination of Rust, Python, and web technologies. The architecture leverages Rust for performance-critical hardware interaction and a web server, while Python is employed for higher-level control logic and bridging the gap between the Rust hardware interface and the web application. A web interface, built with Rust's Rocket framework, provides a graphical user interface (GUI) for configuration, control, and monitoring. Bluetooth communication is also part of the system.

**Key Functionalities and Components:**

*   **Hardware Control (LEDs):** The core function is controlling Raspberry Pi hardware components, with initial focus on LEDs. Users can select different LED display modes (demo, rainbow, cosmic radiation map) and adjust settings for each mode.
*   **Rust Library (rpi\_python/raspberry\_pi\_lib):** Provides low-level hardware access. Uses the `rppal` crate for GPIO control, enabling functions like `blink_led`, `set_high`, and `send_pwm`.
*   **Python Interface (RustPyLib/Lib/rpipy.py):** Acts as a bridge to the Rust library, allowing Python scripts to call Rust functions for hardware control. Uses `cpython` to create a Python extension module. Also handles some Python-specific tasks like getting the CPU temperature using OS commands.
*   **Web Server (rpi\_web):** A Rust-based web server using the Rocket framework to provide a GUI for controlling the hardware. Includes user authentication and WebSocket handling for real-time interaction.
*   **Bluetooth Communication:** The system includes bluetooth commmunication, but primarily used for Raspberry Pi system information. Actual LED control is achieved via serial communication.

**Architectural Insights:**

*   **Layered Architecture:**
    *   *Hardware Layer (Rust):* Directly interacts with Raspberry Pi hardware.
    *   *Abstraction Layer (Python):* Provides an interface for interacting with the hardware layer.
    *   *Web Server Layer (Rust/Rocket):* Handles user interaction and controls the Python layer.
*   **Polyglot Approach:** Rust for performance and low-level control; Python for rapid development and easier integration.
*   **Asynchronous Operations:** Uses threads in Rust and asynchronous I/O for network connections in the web server to improve responsiveness.
*   **Client-Server Model:** Web browsers act as clients, and the Rocket web server as the server. The Python script is a "worker" process.
*   **Event-Driven Communication:** WebSocket communication enables real-time updates from the Python control script or Raspberry Pi to clients.

**Notable Design Patterns:**

*   **Adapter Pattern:** The Python library `rpipy.py` acts as an adapter, translating calls from Python into calls to the Rust library.
*   **Facade Pattern:** The `rpipy.py` module provides a simplified interface to the more complex Rust hardware library.
*   **Modularity:** Organized into separate modules (RustPyLib, WebServer, ScriptControl).
*   **Model-View-Controller (MVC):** Loosely followed, with templates as the View, Rust structs as the Model, and handlers in `main.rs` as the Controller.
*   **Singleton:** The `ServerState` struct with a mutex around `logged_in_user` limits the server to a single active user session.
*   **Observer:** WebSocket implementation allows real-time updates to all connected clients.

**Security Considerations:**

*   Basic login functionality lacks robust security measures (password salting/hashing, input validation, protection against XSS attacks).
*   Storing passwords in `WebServer/users.txt` is highly insecure.
*   Hardcoded IP addresses and ports.
*   Hardcoded key values and socket connections.
*   Authentication issues and single user capability.

**Areas for Improvement:**

*   Comprehensive error handling, logging, and user-friendly error messages.
*   Externalize configuration (IP addresses, ports) to a configuration file.
*   Improve security measures (password hashing, input validation, XSS protection, use a database).
