## Earth Defender: A Distributed Real-Time 3D Game - Architectural Overview

Earth Defender is a distributed soft real-time 3D single/multiplayer game built using Erlang/OTP for the server-side and Three.js for the client-side. The game's objective is to defend Earth from incoming meteorites using a spaceship.

**Key Functionalities:**

The game offers both singleplayer and multiplayer game modes, with the following key features:

*   Real-time 3D rendering using Three.js.
*   Collision detection between spaceships, meteorites, and Earth.
*   Scoring and health tracking.
*   Real-time client-server communication via WebSockets.
*   Fault tolerance and distributed architecture on the server-side (Erlang/OTP).
*   Dynamic room management for multiplayer games, including reconnection logic.
*   Intuitive spaceship controls for movement and shooting.
*   Dynamic asteroid spawning and movement patterns.
*   Game state synchronization between server and clients.

**Client-Side Architecture (Three.js):**

The client-side utilizes Three.js for rendering and handles the user interface and game logic. Key components include:

*   `Game`: The core class managing the game loop, rendering, scene initialization, camera setup, and game element interactions.
*   `GameClient`: Handles WebSocket communication with the Erlang server for multiplayer functionality.
*   `GameDOMHandler`: Manages interactions with the HTML DOM, updating the display with game information (life, score, players) and handling UI elements (game mode selection, room joining).
*   `GameElements`: Provides classes (factories) for creating game objects like Earth, Sun, Moon, SpaceShip, Bullet, and Meteorite using Three.js.
*   `TextureLoader`: A singleton class that caches textures to improve performance.
*   `Light`: Defines lighting parameters for the scene.
*   Specific classes for game objects: `Earth`, `Sun`, `Moon`, `SpaceShip`, `Bullet`, and `Meteorite`, defining their visual and physical properties.

**Server-Side Architecture (Erlang/OTP):**

The server-side, built with Erlang/OTP, emphasizes fault tolerance, concurrency, and scalability. It employs a master-slave architecture with automatic failover, providing a robust multiplayer experience. Key Erlang modules include:

*   `slave_handler.erl`: Handles slave/replica server functionality.
*   `websocket_handler.erl`: Handles WebSocket requests from clients.
*   `player.erl`: Manages individual player state.
*   `utils.erl`: Contains utility functions such as UUID generation and logging.
*   `local_rooms_state.erl`: Manages the state of game rooms.
*   `room.erl`: Manages the game logic for individual rooms.
*   `earth_defender_sup.erl`: Supervisor module responsible for managing server processes.
*   `monitor_mesh.erl`: Monitors the Erlang mesh network for node failures.
*   `earth_defender_app.erl`: Application module that starts the Cowboy web server and the supervisor.
*   Configuration files: `config.hrl` (defines configuration constants) and `room_state.hrl` (defines the room state record).
