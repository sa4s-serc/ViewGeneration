QAPM monitors several key aspects of application performance:

*   **CPU Usage:** Monitors CPU utilization.
*   **Battery Consumption:** Monitors battery usage.

1.  **QAPM Library (qapm-lib/):** The core component responsible for data collection, storage, and reporting. It’s designed with a modular approach, separating concerns into distinct packages. This library houses the core monitoring logic and data management functions.
2.  **QAPM Gradle Plugin (buildSrc/):** Facilitates the AOP injection of monitoring code during the application's build process. The plugin uses ASM bytecode manipulation to achieve this without directly modifying the application's source code.
