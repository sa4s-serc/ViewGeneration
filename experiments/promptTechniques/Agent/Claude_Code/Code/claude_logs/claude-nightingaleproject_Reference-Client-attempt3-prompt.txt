## Nightingale Project Reference Client Analysis

This report summarizes the architecture, key functionalities, and important components of the Nightingale Project Reference Client. The client, built using .NET, C#, and React, serves as a reference implementation for exchanging mortality data between vital records jurisdictions and the NCHS (National Center for Health Statistics) using FHIR (Fast Healthcare Interoperability Resources) messaging. It facilitates the submission of VRDR (Vital Records Death Reporting) FHIR messages, ensures reliable delivery through acknowledgements and retries, and handles the retrieval of message responses.

**Key Functionalities and Components:**

*   **VRDR Message Handling:** The client is responsible for creating, sending, and processing various VRDR message types, including Submission, Update, Void, Alias, Acknowledgement, Coding, Extraction Error, and Status messages.
*   **Reliable Delivery:** Implements mechanisms for acknowledgements and retries, guaranteeing successful message delivery to the NVSS API server.
*   **NVSS API Interaction:** Establishes communication with the NVSS API server for submitting records and retrieving corresponding responses.
*   **Database Persistence:** Utilizes a PostgreSQL database to store messages, responses, and client state information.
*   **API Endpoints:** Exposes REST API endpoints allowing jurisdictions to submit VRDR records and query the status of submitted messages.
*   **User Interface:** Provides a React-based user interface that enables users to upload records and monitor message status.
*   **Configuration:** Leverages `appsettings.json` for configurable settings, including API endpoints, authentication credentials, resend intervals, and polling intervals.
