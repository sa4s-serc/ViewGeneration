## Analysis of `kryptokrauts_ae-prediction-cards` Repository

This report summarizes the architecture, key functionalities, and components of the `kryptokrauts_ae-prediction-cards` repository, a decentralized prediction market platform built on the Aeternity blockchain. The core concept involves predicting future asset prices by renting Non-Fungible Tokens (NFTs) representing potential outcomes (higher or lower). The longest renter of the NFT corresponding to the correct outcome ultimately becomes the NFT owner. The system comprises a smart contract, backend services (oracle and prediction processing), and a React-based frontend for user interaction.

**Key Functionalities:**

*   **Prediction Creation:** Users define predictions by specifying the asset, target price, start/end times, and outcome-related NFT images.
*   **NFT Renting:** Users rent NFTs representing price outcomes, depositing funds and setting a rental price per millisecond. The highest bidder secures the rental.
*   **Oracle Integration:** An oracle service provides the actual asset price at the prediction's end, determining the winning outcome.
*   **Automated Prediction Processing:** A dedicated service automatically queries the oracle and processes prediction results based on the oracle's response and NFT renting durations.
*   **Reward Claiming:** Renters of the winning NFT claim rewards proportionally to their holding time.
*   **NFT Ownership Transfer:**  The longest renter of the winning NFT at the prediction end becomes the new NFT owner.
*   **Frontend Interaction:** A React frontend provides a user interface for creating predictions, renting NFTs, viewing holdings, and claiming rewards.

**Core Components:**

*   **Smart Contract (`contract/PredictionCards.aes`):** This Aeternity smart contract, written in Sophia, is the heart of the system. It governs prediction creation (`create_prediction`), NFT renting (`rent_nft`, `deposit_to_nft`), oracle interaction (`ask_for_winning_option`, `process_oracle_response`), outcome determination, and reward distribution (`claim`). *   **Aeternity Blockchain Integration:** The platform leverages the Aeternity blockchain for trustless execution, data storage (predictions, NFT ownership), and smart contract logic. It uses AENS names for simplified contract access.
*   **Backend Services:**

    *   **Process Prediction Service (`process-prediction-service/src/main/java/...`):** Monitors the blockchain for completed predictions, triggers oracle queries, and processes the resulting data to determine winners.
