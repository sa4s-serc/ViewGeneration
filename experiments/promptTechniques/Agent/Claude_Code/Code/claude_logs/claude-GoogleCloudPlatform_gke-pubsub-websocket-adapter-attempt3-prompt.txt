This report summarizes the architecture, core functionality, and key components of the `gke-pubsub-websocket-adapter` repository. **Core Functionality and Architecture:**

The primary function of the adapter is to bridge Google Cloud Pub/Sub messages to WebSocket clients, enabling real-time data streaming without direct client interaction with the Pub/Sub API. The adapter is designed for deployment on GKE, leveraging its autoscaling and load balancing capabilities. This deployment features a microservices architecture, comprising multiple containers, each with a specific purpose. The stateless design of the clusters simplifies scaling and management.

The core components are:

*   **`pulltop`:** A CLI tool for managing Pub/Sub subscriptions and pulling messages from topics.
*   **`websocketd`:** A command-line tool that serves WebSocket clients, exposing Pub/Sub messages as a WebSocket endpoint.
*   **Ephemeral POSIX Storage:** A memory-mapped filesystem provides temporary storage of recent messages within the container. This acts as a buffer for new clients and decouples WebSocket connections from the Pub/Sub subscription, improving responsiveness.
*   **Cron Job:** A scheduled job rotates the output JSON file.

The system leverages Workload Identity for secure authentication with Pub/Sub.
