**Core Functionality and Components:**

Activiti 7 leverages a microservices architecture built for scalability and resilience. Key components include:

*   **Activiti Core**: The core process engine library, embedded within Java applications, responsible for executing BPMN processes.

*   **Activiti Cloud Runtime Bundle**:  A stateless, scalable service acting as the core of Activiti Cloud, responsible for executing process definitions and interacting with external systems through connectors.

*   **Activiti Cloud Connectors**: Facilitate bi-directional integration between processes and external services. They decouple the process engine from external systems, enabling scalability and easier maintenance.

*   **Activiti Cloud Query Service**: Provides read-only, efficient access to process and task data, aggregating data from Runtime Bundles.

*   **Activiti Cloud Audit Service**: Consumes events from Runtime Bundles and persists them for auditing and compliance purposes.

*   **Activiti Cloud Notification Service**: Enables real-time data updates using GraphQL subscriptions and WebSockets.

*   **Gateway**: A single entry point for interacting with Activiti Cloud services.

*   **Identity Management / Single Sign-On (SSO)**: Leverages Keycloak for authentication, authorization, and user management.

*   **Spring Cloud Integration**: Integrates with Spring Cloud Stream for asynchronous communication, Spring Cloud Kubernetes for Kubernetes integration, and Spring Cloud Gateway for routing.

*   **BPMN Modeling Application**:  A UI for designing and exporting BPMN process definitions.

*   **Helm Charts**: Used for deploying Activiti Cloud components to Kubernetes.

*   **Jenkins X**: Used for CI/CD pipeline automation.

**Architectural Design and Patterns:**

Activiti Cloud's architecture is inherently cloud-native, employing several important design principles:

*   **Cloud-Native Design**: The platform is designed specifically for cloud environments, prioritizing scalability, resilience, and independent deployment of services.

*   **Microservices Architecture**: The platform is decomposed into independent microservices that communicate over a network, allowing for independent scaling and deployment.

*   **Stateless Services**:  Runtime Bundles are stateless to facilitate horizontal scaling and improved resource utilization.

*   **Event-Driven Architecture**: Components communicate asynchronously using events, primarily through Spring Cloud Stream, decoupling services and enhancing scalability.

*   **Kubernetes as the Primary Platform**: Kubernetes is the target deployment platform, leveraging its features for orchestration, scaling, and service discovery.

*   **GitOps**: Utilizes Jenkins X and Git repositories to manage and automate deployments, leveraging version control for infrastructure and application configurations, streamlining deployments and facilitating rollbacks.

The platform also utilizes several notable design patterns:

*   **Bill of Materials (BOM)**: Used for managing dependencies and ensuring consistency across projects.
*   **Cloud Connectors as Service Task Implementations**: Connectors are used to implement service tasks, enabling interaction with external systems.
*   **Command Pattern:** Payloads represent commands to be executed by the engine.
*   **Event Listeners**: Used to react to process engine events and update the state of other services (e.g., Query Service, Audit Service).
*   **Dependency Injection (Spring Framework)**: Used extensively to configure and wire components together, promoting modularity and testability.
