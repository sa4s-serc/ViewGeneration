## Piattaforma Notifiche (PN) Infrastructure Analysis Report

This report summarizes the infrastructure defined in the `pagopa_pn-infra` GitHub repository, focusing on key functionalities, components, architecture, and design patterns. The repository primarily utilizes AWS CloudFormation for Infrastructure as Code (IaC), enabling automated, repeatable, and version-controlled deployments of the "Piattaforma Notifiche" (PN) project, a notification platform likely used by the Italian government ("PagoPA").

**Key Functionalities and Components:**

*   **Infrastructure as Code (IaC):** CloudFormation templates define and manage the entire AWS infrastructure.
*   **API Gateway and Lambda Functions:** Defines mock APIs for Proof of Concept (PoC) purposes (e.g., address book) and components for processing notifications (e.g., webhooks), exposed via API Gateway with features like custom domains, authorizers, usage plans and WAF integration. Also implements reverse proxy functionality through Lambda, routing requests based on URL paths.
*   **Microservice Support:** Defines resources for deploying microservices including ECS clusters, task definitions, and service configurations using Fargate, often with sidecar containers for observability via `aws-otel-collector`.
*   **Event-Driven Architecture:** EventBridge (formerly CloudWatch Events) is used to route events between microservices and trigger Lambda functions based on state changes. Extensive use of SNS topics and SQS queues for asynchronous communication.
*   **Monitoring and Alarms:** Extensive monitoring setup using CloudWatch, including alarms for ECS tasks, API Gateway latency, Lambda errors, Kinesis stream iterator age, SSL certificate expiration, and SQS queue sizes.  Integrates with SNS for notifications (likely to Slack). Includes mechanisms for composite metrics generation based on alarm status.
*   **Logging and Auditing:** Comprehensive logging setup, sending logs to S3 buckets, CloudWatch Logs and Kinesis streams for analysis. Includes CloudTrail trails for auditing Lambda function invocations. Features include centralized logging, log masking, and violation checking.
*   **Data Management:**
    *   Defines backup plans for DynamoDB tables using AWS Backup.
    *   Includes scripts for data migration between DynamoDB tables, with potential sanitization/transformation.
    *   Implements CDC (Change Data Capture) processing, using Athena to update CDC parsed cache.
    *   Configures S3 buckets, KMS keys, and Kinesis streams for logging and data storage.
*   **Celonis Integration:** Defines resources for exporting data to Celonis, a process mining platform, including S3 buckets, IAM users with limited permissions, key rotation, and a CodeBuild project to copy data.
*   **Queue-based Communication:** Uses SQS queues extensively for asynchronous communication between microservices. Uses standard naming convention for queue identification.
*   **Authentication and Authorization:** Employs Cognito user pools for authentication and authorization, integrated with API Gateway. Also uses IAM roles with least-privilege permissions, and Security Groups to control network access.
*   **Parameter and Secret Management:** AWS Systems Manager Parameter Store and Secrets Manager are used to store configuration data and sensitive information. Includes scripts for backing up and cleaning up SSM parameters and Secrets Manager secrets.
*   **Cost Optimization**: Contains templates to stop EC2 instances and ECS services during off-peak hours, to reduce costs.
*   **Resource Tagging**: Implements a mechanism for automatically tagging AWS resources.
