The application adopts a **component-based architecture**, promoting reusability and maintainability. **Vuex** is used for centralized state management, while **Vue Router** handles navigation. The `api/` directory abstracts backend API interactions, enhancing modularity.

Notable design patterns include:

*   **Module Pattern:** Utilized within Vuex for organizing state, getters, mutations, and actions.
*   **Component Pattern:** Emphasizes the use of reusable Vue components.
*   **Simplified API Gateway:** `api/index.js` centralizes Axios configuration and interceptors.
*   **Implicit Observer Pattern:** Vue's reactivity system enables components to react to Vuex store changes.
*   **Singleton (EventBus):** `eventBus.js` facilitates communication between unrelated components.
