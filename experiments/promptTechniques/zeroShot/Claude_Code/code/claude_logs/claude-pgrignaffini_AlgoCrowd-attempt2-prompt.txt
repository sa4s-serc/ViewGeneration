## Decentralized Crowdfunding Platform Analysis: Report

This report provides an overview of a decentralized crowdfunding platform built on the Algorand blockchain. The platform comprises a client-side Next.js application and a server-side component (Express.js) that interacts with an SQLite database and the Algorand blockchain using PyTeal-based smart contracts. Users can create, fund, and claim refunds for crowdfunding projects, while project creators can collect funds if successful.

**Key Functionalities:**

*   **Project Creation:** Users define projects with details like name, goal, start/end dates, description, and an image URL, triggering Algorand smart contract deployment.
*   **Project Funding:** Users contribute funds to active projects, with a platform fee deducted from each contribution.
*   **Refunds:** Contributors can claim refunds if a project fails to reach its funding goal.
*   **Fund Collection:** Project creators can collect funds if the project reaches its goal.
*   **Account Connection:** Users connect to the platform via AlgoSigner.

**Architecture & Components:**

The architecture employs a frontend-backend separation, with the Next.js frontend consuming data from the Express.js backend via RESTful API calls. The core business logic resides within Algorand smart contracts, ensuring transparency and decentralization.

**Frontend (Client/):**

*   Utilizes Next.js for a dynamic user interface.
*   **`pages/`:** Contains Next.js pages for various routes (homepage, create project, project details, user area).
*   **`components/`:** React components (e.g., `Header`, `Footer`, `Project`, `CreateProjectForm`, `FundProject`, `DisplayProjects`) comprise the UI.
*   **`APIs/API.js`:** Defines API functions for interacting with the backend.
*   **`context/AppContext.js`:** Manages global application state, including the Algod client. Provides this client to other components using the Provider Pattern.
*   **`constants/`:** Stores constants like Algod token, server base URL, platform fee, and global state keys. Stores the compiled TEAL code for the approval and clear programs.
*   **`utils/`:** Contains utility functions for contract and transaction operations (`ContractOperations.js`, `TxOperations.js`), date formatting, and other helper functions.

**Backend (Server/):**

*   **`server.js`:** Defines the Express.js server and REST API endpoints for creating applications, funding applications, and retrieving application data. Route handlers orchestrate logic for incoming HTTP requests.
*   **`db.js`:** Manages the SQLite database connection.
*   **`dao.js`:** Implements the Data Access Object (DAO) pattern, providing an abstraction layer for database interaction and isolating the rest of the application from specific database implementations.

**Smart Contracts (crowdfunding-demo/crowdfunding/):**

*   **`contracts.py`:** Defines the TEAL logic for approval and clear state programs using PyTeal, governing funding, refunds, and fund collection.
*   **`operations.py`:** Contains functions for creating, setting up, funding, refunding, and closing crowdfunding projects.  The `getContracts` function can be viewed as an implicit factory responsible for creating and returning the compiled TEAL contracts.
*   **`util.py`:** Contains utility functions for interacting with the Algorand blockchain.

**Key Files & Configuration:**

*   **`README.md`:** Provides an overview of the project, usage instructions, and development setup.
*   `client/netlify.toml`: Configuration for Netlify deployment.
*   `client/next.config.js`: Next.js configuration.
*   `client/tailwind.config.js`: Tailwind CSS configuration.
*   `crowdfunding-demo/crowdfunding/testing/`: Provides testing resources and setup scripts.

**Design Patterns:**

*   **Data Access Object (DAO):** `dao.js` provides a clean interface for interacting with the SQLite database.
*   **Factory Pattern (Implicit):** `getContracts` in `crowdfunding/operations.py` creates and returns compiled TEAL contracts.
*   **Provider Pattern:** `AppContext.js` acts as a provider for the algodClient.

**Additional Information:**

*   Timestamps are handled for start and end dates, converting them to UNIX timestamps.
