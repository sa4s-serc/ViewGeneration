## Integrated Analysis: LwM2M and Blockchain System

This report synthesizes the analysis of a GitHub repository implementing a system that integrates LwM2M (Lightweight M2M) with a blockchain, primarily Ethereum, to enhance reliability, auditability, and security of IoT device management and anomaly detection. The system comprises an Angular-based anomaly detection application, a Spring Boot backend, Solidity smart contracts, and LwM2M components.

**Core Functionality & Components:**

The system centers around three main functionalities: User and Client management, Anomaly detection, and LwM2M device interaction.

*   **Anomaly Detection & Management:**
    *   An Angular-based web UI allows monitoring of LwM2M clients and displays detected anomalies. The application fetches client data, triggers temperature analysis, and reports anomalies to the backend for blockchain storage.
    *   The Spring Boot backend exposes a REST API for the Angular frontends and interacts with smart contracts to manage users, clients, and anomaly data using the Web3j library for Ethereum communication.
    *   The `AnomalyStore` smart contract records detected anomalies with timestamps, affected endpoints, emergency levels, and encoded data, enabling querying based on various criteria.
*   **User & Client Management:**
    *   `UserStore` smart contract manages user accounts, roles, authentication through adding, updating and validating users. Authentication is implemented using JWTs.
    *   `ClientStore` smart contract stores and manages client (device) registration information, including endpoint details, bootstrap server configurations, and LwM2M server configurations.
*   **LwM2M Implementation:**
    *   **LwM2M Server:** Monitors the environment through registered clients, handles client-server interaction over CoAP, and provides a secure REST API. Leverages the Eclipse Leshan library.
    *   **Bootstrap Server:** Manages client credentials and bootstrapping to the LwM2M server by retrieving security information from the blockchain.
    *   **LwM2M Client:** Simulates IoT devices measuring physical parameters (e.g., temperature). Includes standard LwM2M Security, Server, and Device objects.
*   **Evaluation:** Jupyter Notebooks and the Plotly library used to visualize performance metrics.

**Key Files & Their Roles:**

The repository is structured with distinct directories for different components:

*   **`anomalyDetectionApp/`:** Contains the Angular anomaly detection application frontend. It includes components for UI elements (ClientComponent, AnomalyComponent, LoginComponent), services for backend API interaction (ClientService, AnomalyService, LoginService, AuthService), and routing configuration.
*   **`mainApp/`:** Contains both the Spring Boot backend (`MainAppBackEnd/`) and a management Angular frontend (`MainAppFrontEnd/`). The backend includes REST API controllers (ClientController, AnomalyController, LoginController, UserController), services for business logic and blockchain interaction (ClientService, AnomalyService, UserService, BlockchainService), Web3j wrapper classes for smart contracts (`wrappers/`), and data model classes (`beans/`).
*   **`smartContracts/`:** Contains the Solidity smart contracts (`UserStore.sol`, `ClientStore.sol`, `AnomalyStore.sol`).
*   **`leshan/`:**  Contains the LwM2M components: Server, Bootstrap Server, and Client. It utilizes Eclipse Leshan library, Web3j, and JWT libraries.

**Architectural Insights:**

*   **Layered Architecture:** Front-end, back-end, and smart contract layers promote separation of concerns and modularity.
*   **Microservices:** The separate services for LwM2M functionality and blockchain interaction suggest a microservices orientation.
*   **Decentralized Data Storage:** Critical information and security configurations are stored on the blockchain, enhancing security and auditability.
*   **Event-Driven (Potentially):**  The anomaly detection suggests an event-driven approach.
*   **Security:** Authentication implemented using JWTs. Smart contracts include access control mechanisms like `onlyOwner` and `onlyWhitelist`.

**Notable Design Patterns:**

*   **REST API:** The Spring Boot backend exposes RESTful endpoints for communication.
*   **Data Transfer Objects (DTOs):** `beans` directory defines DTOs for data exchange between layers.
*   **Service Layer:** Encapsulates business logic and blockchain interactions.
*   **Remote Proxy:** Web3j-generated wrapper classes act as remote proxies for smart contracts.
*   **Middleware (JWT):** JWTs are used as middleware to secure API endpoints.
*   **Adapter Pattern:** `ClientStoreService` and `BlockchainBootstrapStore` adapt Leshan security interfaces to the blockchain.
*   **Factory Pattern:** Used in generating plots for performance evaluation.
