*   **Localization:** Support for English and Korean translations using `locales/en.js` and `locales/ko.js`.
*   **Markdown Rendering:** Syntax highlighting powered by `public/scripts/highlight.js` & `public/scripts/highlight-c.js` and markdown text converted to HTML by `./public/scripts/showdown.js`, with a markdown preview available using `public/scripts/preview.js`. Integrated with CodeMirror 2 editor on the client side.

**II. Architecture and Important Files:**

The application utilizes a modular design with separation of concerns:

*   **Server-Side (Node.js/Express):** `app.coffee` is the main application file, setting up the Express server, Socket.IO, middleware, and routes.
*   **Client-Side (JavaScript):** jQuery is used extensively for DOM manipulation. Bootstrap provides styling and layout.
*   **Data Storage:** Git repository serves as the database, offering version control and data persistence.
*   **Configuration:** `config.js` manages application settings.
*   **Routing:**  Express routes are defined in `routes/index.js` handling user requests.
*   **Core Logic:** The `lib/` directory contains core functions and class definitions, including:

*   **Testing:**  Uses Mocha tests (`gitfs.test.js`, `wiki.test.js`, etc.).  `.travis.yml` configures automated testing.  `runtest.js` runs mocha tests and `Makefile` defines tasks for testing.
