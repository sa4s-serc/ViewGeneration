This report summarizes the architecture, functionality, and key components of the `ActiveX.NET` framework, which enables the creation of Out-Of-Proc COM servers using C#. The framework leverages the Managed Extensibility Framework (MEF) for dynamic loading of COM objects implemented in separate DLLs (plugins) into a central EXE COM server.

**Key Functionalities:**

*   Implementation of Out-Of-Proc COM servers using C#.
*   Dynamic loading of COM objects from plugin DLLs via MEF.
*   Automatic COM object registration and unregistration.
*   Reference counting and object lifetime management.
*   Support for STA and MTA threading models.
*   Enabling .NET WinForms/UserControls inside COM Objects.

**Architectural Overview:**

The architecture is built upon a plugin model, facilitating the addition of new COM objects without modifying the core server application. MEF is central to this, providing dependency injection and dynamic discovery of `IActiveXServer` implementations. The default threading model is STA, with an MTA option available for parallelism. Crucially, the design incorporates COM object lifecycle management, handling registration, unregistration, and reference counting.
