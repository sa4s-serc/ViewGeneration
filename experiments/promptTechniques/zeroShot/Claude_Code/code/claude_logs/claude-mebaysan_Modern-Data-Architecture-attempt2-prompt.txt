**Report:**

This repository presents a modern data architecture solution, orchestrating data extraction, transformation, warehousing, and visualization using Airbyte, DBT (Data Build Tool), Apache Superset, and PostgreSQL. It's designed for easy deployment and management through Docker and Docker Compose, employing git submodules to promote code reuse and modularity. The primary use case demonstrated is the processing and analysis of the Kaggle Brazilian E-Commerce dataset.

**Key Functionalities and Components:**

*   **Data Extraction:** The repository offers two methods:
    *   **Airbyte Integration:** Airbyte serves as the primary data extraction tool, designed to ingest data from a local CSV directory (containing the Kaggle dataset).
    *   **Python Script (Alternative):** A `Source/script.py` Python script provides an alternative method for loading CSV data directly into PostgreSQL. This script relies on environment variables for database connection details, defined in a `.env` file.

*   **Data Warehousing (PostgreSQL):** PostgreSQL functions as the data warehouse, storing both raw and transformed data. The `Postgres-DWH/docker-compose.yml` file facilitates the setup of PostgreSQL and pgAdmin using Docker Compose. Volume mounts ensure data persistence. A secondary Docker Compose file (`Postgres-DWH/docker-compose-nginx.yml`) integrates Nginx as a reverse proxy.

*   **Data Transformation (DBT):** DBT is employed for data transformation and modeling within the PostgreSQL data warehouse. The DBT project configuration resides in a submodule (`Modern-Data-Architecture-DBT`).

*   **Data Visualization (Apache Superset):** Apache Superset provides the data visualization layer, enabling the creation of dashboards and visualizations based on the transformed data. It is set up via a submodule (`Superset-Production-Environment`).

**Architectural Overview:**

The repository adheres to a layered architecture, encompassing:

1.  **Data Ingestion:** Handled primarily by Airbyte, with a Python script as a backup.
2.  **Data Warehousing:** PostgreSQL serves as the central data repository.
3.  **Data Transformation:** DBT performs data modeling and transformations.
4.  **Data Visualization:** Superset visualizes the transformed data.

The project leverages Docker to containerize each major component (Airbyte, DBT, Superset, PostgreSQL), allowing for independent deployment and management. Docker Compose orchestrates these containers, defining the infrastructure as code (IaC) and simplifying the setup process. The use of git submodules for Airbyte, DBT, and Superset promotes a modular design.
