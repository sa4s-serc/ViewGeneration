**Architectural Highlights:**

The system adopts a microservices architecture for modularity and scalability. Key architectural elements include:

*   **Microservices Architecture:** Separating functionalities into independent, deployable services.
*   **API Gateway:** A single entry point for all frontend requests, routing them to the appropriate backend service and enforcing authentication. This functions as a simplified Backend for Frontend (BFF) pattern.
*   **Asynchronous Communication:** Redis Pub/Sub enables asynchronous communication between the gateway and tracking services, improving performance and decoupling.
