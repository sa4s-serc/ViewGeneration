## Solenya Framework: A Comprehensive Overview

Solenya is a front-end web framework designed for conceptual simplicity and seamless integration with TypeScript. It employs a component-based architecture, leveraging a virtual DOM for efficient UI updates and offering features like serialization, routing, validation, and styling. Solenya aims to simplify web application development by promoting state management within components and treating views as pure functions of that state. It encourages component composition and provides utilities for form building, user input handling, time-travel debugging, and hot module reloading.

**Key Functionalities:**

*   **Component-Based Architecture:** Facilitates building UIs through reusable, modular components.
*   **Virtual DOM:** Enables efficient UI updates by minimizing direct DOM manipulations.
*   **State Management:** Encourages centralized state within components for predictable behavior.
*   **Routing and Navigation:** Provides mechanisms for navigating between different application views.
*   **Form Building and Validation:** Simplifies form creation and data validation.
*   **Serialization/Deserialization:** Supports persistence and time travel through state serialization.
*   **Styling with Typestyle:** Integrates with `typestyle` for type-safe CSS-in-JS styling.
*   **Lifecycle Events:** Offers hooks for interacting with the DOM during component lifecycle stages.
*   **Time Travel Debugging & Hot Module Reloading:** Enhances development workflow with advanced debugging and live updates.

**Core Components:**

*   **`Component`:** The base class for all UI components. It manages component state, the rendering function (`view`), and update cycles. Dependency Injection is facilitated via properties like `app` and `parent`.
*   **`App`:** The entry point for the framework. Responsible for managing the root component, rendering, serialization, and enabling time travel features.
*   **`Router`:** Handles navigation and routing between different components within the application.
*   **`Validator`:** Provides validation capabilities leveraging the `class-validator` library, with Decorator Pattern implementation.
*   **`Storage`:** Manages serialization and persistence of component state to local storage.
*   **`TimeTravel`:** Implements time travel debugging, allowing developers to step through component state history.
*   **HTML Helper Functions:** A set of functions (e.g., `div`, `span`, `input`) to create virtual DOM nodes in a type-safe manner.
*   **Widget Functions:** Functions (e.g., `inputText`, `inputNumber`) to simplify the creation of common form input elements.
