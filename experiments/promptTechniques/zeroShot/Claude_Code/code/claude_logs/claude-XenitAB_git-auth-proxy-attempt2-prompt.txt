ARCHITECTURAL SUMMARY:
**XenitAB `git-auth-proxy` Analysis Report**

The `git-auth-proxy` project provides multi-tenant access to Git repositories (GitHub and Azure DevOps) within Kubernetes environments by acting as a proxy server. It introduces an authentication and authorization layer between Git clients and the Git providers. The core concept involves generating static tokens for specific repositories, storing them as Kubernetes secrets within namespaces that require access. The proxy then validates these tokens against the requested repository during clone or API requests.

**Key Functionalities and Components:**

*   **Proxy Server:** Intercepts Git requests, authenticates them, and forwards them to the appropriate Git provider.
*   **Token-Based Authentication:** Generates unique tokens for each repository/namespace combination for authentication.
*   **Authorization Layer:** Determines if a token possesses the necessary permissions for the requested repository.
*   **Configuration Loading:** Reads a JSON configuration file, defining organizations (GitHub or Azure DevOps), repositories, and associated Kubernetes namespaces.
*   **Kubernetes Secret Management:** Manages Kubernetes secrets containing the tokens, including creation, updates, and deletion, synchronizing tokens by listening for Secret changes in the cluster.
*   **Metrics Endpoint:** Exposes Prometheus-compatible metrics.

*   **Configuration-Driven:** Proxy behavior is determined by configuration, allowing flexible management of organizations, repositories, and permissions.
*   **Kubernetes-Native:** Utilizes Kubernetes secrets, integrates with the Kubernetes API, and is deployable via Helm.
*   **Multi-tenancy:** Supports multiple namespaces accessing repositories with fine-grained permissions.

**Notable Design Patterns:**

*   **Factory Pattern (Implicit):** The `NewAuthorizer` function functions as a factory, creating an `Authorizer` instance based on configuration, while the `Provider` interface allows for Git provider specific implementations.
