## LouisvilleMetro_WazeCCPProcessor Repository Analysis

This report summarizes an analysis of the `LouisvilleMetro_WazeCCPProcessor` repository, a cloud-based, open-source solution for processing and analyzing data from the Waze Connected Citizens Program (CCP). **Key Functionalities and Components:**

The repository is structured around four key components:

*   **Data Ingestion:** The `waze-data-download` Lambda function periodically fetches data from the Waze CCP data feed URL and stores it as JSON files in an S3 bucket (acting as a data lake).
*   **Data Processing:** A suite of Lambda functions (`waze-data-process`, `waze-data-alerts-processing`, `waze-data-jams-processing`, `waze-data-irregularities-processing`) processes the raw JSON data. This involves parsing, transforming it into a relational schema, and storing it in an Aurora PostgreSQL database. SQS queues manage the processing workflow for reliable delivery, and SNS topics provide notifications about data retrieval and processing status. The `waze-data-process` function orchestrates the processing of Waze data files, distributing the workload to other specialized lambdas.
*   **Database Management:** An Aurora PostgreSQL database stores the processed Waze data. The `waze-db-initialize` Lambda function initializes and manages the database schema using SQL scripts, considering schema updates and versioning. The `./code/lambda-functions/waze-data-process/src/db/*` directory contains database access logic for interacting with the database.
*   **API Access:** The API Gateway exposes the `waze-data-api` Lambda function, providing access to stored Waze data (jams, alerts, irregularities). Functionality includes data filtering, pagination, and coordinate inclusion. Data models for API requests and responses, including validation logic, are defined in `./code/lambda-functions/waze-data-api/src/api-models/*`.
