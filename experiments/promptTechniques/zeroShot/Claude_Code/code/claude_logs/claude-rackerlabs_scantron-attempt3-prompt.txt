This report consolidates the analysis of the Scantron project, a distributed network scanner designed for scheduling, distributing, executing, and processing network scans. **Core Functionality & Components:**

The system's core lies in **scan scheduling**, driven by `django-recurrence` for complex recurrence rules. The `scan_scheduler.py` script is central to this, managing scan instances based on database configurations and scheduling logic.  **Distributed scanning** distributes tasks to engines via a REST API, where engines poll the console for new jobs.

Engines then **execute scans** using `nmap` or `masscan`.  **Result handling** involves engines storing scan results, often on a shared NFS volume, which the console processes. The system employs a **REST API** (built with Django REST Framework) for communication between the console and engines, enabling external systems to schedule and monitor scans. API tokens are used for authentication. An API client (`scantron_api_client.py`) allows programmatic interaction.  The system also supports **email alerts** upon scan completion, errors, or scan diffs.
