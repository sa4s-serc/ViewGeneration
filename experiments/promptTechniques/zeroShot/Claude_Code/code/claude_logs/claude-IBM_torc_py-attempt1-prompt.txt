**Core Functionality & Components:**

The library's core is built around task management, application setup and control, and load balancing.

*   **Task Management:** Provides mechanisms for submitting, executing, and retrieving results from tasks. Key functions include:
    *   `submit()`: Submits tasks asynchronously for distribution across workers, returning a `TaskT` future.
    *   `map()`:  Executes a function in parallel over a sequence of arguments.
    *   `wait()`: Blocks until all child tasks of the current task have completed.
    *   `as_completed()`:  Similar to `wait`, but returns finished child tasks as they complete.

*   **Application Setup & Control:** Manages the lifecycle of the parallel application. Important functions are:
    *   `start()`: Initializes the library, launches the main application task on rank 0, and shuts down after completion.
    *   `init()`: Initializes the core tasking library components.
    *   `launch()`: Launches a function as the primary application task on the MPI process with rank 0.
    *   `shutdown()`: Shuts down the tasking library and releases resources.

*   **Load Balancing:** Automatically distributes tasks across workers using a round-robin approach by default. Also implements a task stealing mechanism to redistribute work from busy to idle processes.

*   **SPMD Support:** `spmd()` executes a function on all MPI processes, enabling a switch to Single Program, Multiple Data (SPMD) execution mode for legacy MPI code.

*   **Callbacks:** Supports user-defined callback functions to be executed upon task completion, facilitating post-processing and reduction operations.

*   **Context Manager:** A `TorcPoolExecutor` allows code to be wrapped in a context to manage a thread pool.

*   **Task Queue:** Each process maintains a `torc_q` task queue to store tasks awaiting execution by worker threads.
*   **Server Thread:** A dedicated `torc_server_thread` manages communication with other processes, receiving tasks, sending results, and handling task stealing requests, facilitating asynchronous operations.
