This report provides an overview of the Emomemo note-taking application, focusing on its key functionalities, architecture, and design patterns. The application is built using Kotlin and leverages several modern Android development tools and best practices.

**Architecture:**

It adheres to Clean Architecture principles, ensuring the domain layer is independent of data and presentation layers. Use cases within the domain layer orchestrate data access through repositories. The architecture is also layered into Data, Domain and Presentation which allows for scalability and ease of updating and debugging.
