ARCHITECTURAL SUMMARY:
## Report: ev2900_OpenSearch_Dashboard_Nginx_Proxy Repository Analysis

This repository provides a solution for securely accessing an OpenSearch dashboard deployed in a private subnet via the public internet using an Nginx proxy server. The solution leverages Infrastructure as Code (IaC) through a CloudFormation template for automated deployment of the necessary AWS resources.

**Functionality:**

The core functionality is enabling secure public access to OpenSearch dashboards hosted within a private subnet. This is achieved by using Nginx as a reverse proxy to forward incoming HTTPS traffic to the OpenSearch dashboard, effectively hiding its internal location and providing a single point of entry. SSL/TLS is employed, utilizing self-signed certificates for encryption between the user and the Nginx proxy server.

**Key Components & Files:**

*   **`opensearch_nginx.yaml` (CloudFormation Template):** Automates the deployment of the entire infrastructure including:
    *   VPC and Subnets (public and private)
    *   Internet Gateway
    *   Security Groups (controlling traffic between the EC2 instance and the OpenSearch domain)
    *   OpenSearch Domain
    *   IAM Role and Instance Profile
    *   EC2 Instance (hosting the Nginx proxy)
    *   Elastic IP
    *   Route Table
*   **`ngnix_config` (Nginx Configuration File):** Defines the proxy rules, SSL settings, and redirection logic. Users must replace placeholders like `<os_domain_url>` with their specific OpenSearch domain URL.
*   **`README.md`:** Provides detailed instructions for deployment and configuration, covering: CloudFormation stack launch, Nginx installation and configuration, SSL certificate creation, OpenSearch dashboard access, and planned future improvements.
*   **`.github/workflows/upload_s3.yml`:** A GitHub Actions workflow automating the upload of the CloudFormation template (`opensearch_nginx.yaml`) to an S3 bucket.
*   **`Read_Me_Architecture/ReadMe_Architecture.drawio`:** A visual representation of the solution's architecture in Draw.io format.

**Architecture:**

The architecture utilizes a tiered approach: A private OpenSearch domain, an Nginx proxy server in a public subnet within the same VPC, and controlled traffic flow via Security Groups. The Nginx instance acts as a reverse proxy, centralizing access and enhancing security. Requests from the public internet are routed to the Nginx proxy, which then forwards them to the OpenSearch dashboard residing in the private subnet.

**Design Patterns:**

*   **Reverse Proxy:**  Nginx shields the internal infrastructure by acting as a reverse proxy.
*   **Infrastructure as Code (IaC):** The CloudFormation template enables repeatable and automated deployments, facilitating infrastructure management.
