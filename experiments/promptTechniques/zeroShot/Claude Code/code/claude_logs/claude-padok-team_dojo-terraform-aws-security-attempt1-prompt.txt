## Terraform Security Dojo Repository Analysis

This report analyzes a GitHub repository containing a Terraform-based exercise (Dojo) designed to teach infrastructure security on AWS. It outlines the key functionalities, components, architecture, and potential areas for improvement.

**Summary:**

The repository provides a hands-on tutorial guiding users through deploying and incrementally securing an AWS infrastructure.  The exercise starts with an insecure setup and progresses through stages of improvement, including environment separation, load balancing, and network filtering using Terraform. A preconfigured VSCode development environment supports user interaction with the Dojo.

**Key Functionalities and Components:**

*   **Infrastructure as Code (IaC):** The core functionality revolves around defining and deploying AWS infrastructure using Terraform. This includes VPCs, subnets, EC2 instances, security groups, and load balancers.
*   **Environment Isolation:** Demonstrates the separation of development ("dev") and production ("prd") environments.
*   **Load Balancing:** Implements load balancing to distribute traffic and enhance security.
*   **Network Security:** Focuses on securing the infrastructure through security groups and IP whitelisting.
*   **Exercise/Tutorial:** The repository is structured as a guided tutorial, providing steps, hints, and solution examples.
*   **VSCode Remote Boxes:** Provides preconfigured development environments via VSCode containers with all necessary tools for completing the Dojo.

**Important Files and Their Roles:**

*   **`LICENSE`:** Apache 2.0 license file.
*   **`README.md`:**  The central documentation hub, providing instructions, exercise steps, and goals.  It guides users through environment setup, Terraform code exploration, and implementing security enhancements.
*   **`iac/main.tf`:**  The primary Terraform configuration file defining AWS resources (EC2 instances, load balancers, security groups) and their configurations. This file is modified during the exercise to improve security posture.
*   **`iac/output.tf`:** Defines Terraform outputs, such as instance IP addresses and SSH connection strings.
*   **`iac/variables.tf`:** Defines input variables used in Terraform configurations, including AWS region, profile, environment, VPC ID, subnet IDs, and IP whitelists.
*   **`iac/network.tf`:** Defines network-related resources, specifically security groups and their ingress/egress rules.
*   **`iac/prd.tfvars` and `iac/dev.tfvars`:** Terraform variable definition files for production and development environments, respectively. These files specify environment-specific values for variables like VPC ID and subnet IDs.
*   **`iac/_providers.tf`:** Specifies Terraform providers (AWS and random) and their configurations, including AWS region and profile.
*   **`iac/_backend.tf`:** Configures the Terraform backend (local), which determines where the Terraform state is stored.
*   **`iac/web/install_apache.sh.tpl`:** A shell script template used as EC2 instance user data to install Apache and create a simple environment-identifying HTML page.
*   **`bootstrap/main.tf`**: Creates VPC and Subnets.
*   **`bootstrap/output.tf`**: Outputs VPC ID, Public Subnets and Private Subnets for each environment.
*   **`bootstrap/env.tfvars.tpl`**: Template to generate environment's `tfvars` file based on bootstrap's output.
*   **`solution/step_*/*`:** Contains example solutions for each step of the exercise.
*   **`vscode-box/*`**: Contains terraform code to provision a VSCode environment.
