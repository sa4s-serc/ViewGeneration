## DevOps Girls Bootcamp: Serverless Todo Application - Repository Analysis

This repository provides materials for a DevOps Girls bootcamp, guiding participants in building a serverless todo application on AWS using both manual configuration ("Click-Ops") and infrastructure as code via the Serverless Framework. The core application allows users to create, read, update, and delete tasks.

**Key Functionalities and Components:**

The application leverages a serverless architecture consisting of the following components:

*   **Todo Application:** Core CRUD functionality for managing todo items.
*   **UI:** A React-based frontend, built with the SEEK Style Guide, providing a user interface to interact with the application. Key components include: `App.js` (main application component), `TodoList.js` (renders the list of todos), `NewTodoForm.js` (form for adding new todos), `config.js` (defines the API endpoint URL), `client.js` (client-side rendering), and `render.js` (server-side rendering).
*   **S3 Bucket:** Hosts the static website (HTML, CSS, JavaScript) of the todo application.
*   **API Gateway:** Acts as the front-end for the Lambda function, routing HTTP requests to the backend logic. Employs a proxy pattern (`/{proxy+}`) forwarding requests to the Lambda function.
*   **Lambda Function:** Contains the application logic for handling API requests (GET, PUT, DELETE) and interacting with DynamoDB.
*   **DynamoDB:** Stores todo data in a NoSQL database.
