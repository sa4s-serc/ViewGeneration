**Core Functionalities and Components:**

*   **`Stalk` (Network Traffic Monitoring and Attack Detection):** Employs the Ryu SDN framework to monitor network traffic, gather flow statistics, and detect potential attacks based on configured traffic thresholds. It analyzes Rx and Tx traffic patterns to identify hosts exceeding threshold values, marking them as potential attackers. `Stalk` also implements basic packet forwarding using `simple_router.py`. It exposes a REST API for mitigation requests.

*   **`Pollen` (Blockchain Integration and Data Storage):** Handles all communication with the private Ethereum blockchain, including attack reporting, retrieval of attack information, and management of contract interactions. It also manages data storage using InfluxDB for traffic statistics and IPFS for attack reports and public keys, employing encryption (RSA) for sensitive data. Solidity smart contracts (`relay.sol`, `autonomous_system.sol`) define the logic for attack reporting and mitigation on the blockchain.

*   **`BloSS` (Mitigation Request Handling and Attack Signaling):** Manages mitigation requests and attack signaling. It also exposes a REST API for reporting attacks and marking them as blocked. `BloSS` sets up API threads, manages blockchain interactions, and schedules tasks.
