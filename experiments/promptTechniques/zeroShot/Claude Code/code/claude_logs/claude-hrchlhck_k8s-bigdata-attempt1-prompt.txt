## Analysis of the `hrchlhck_k8s-bigdata` Repository

This report provides an analysis of the `hrchlhck_k8s-bigdata` GitHub repository, focusing on its functionality, components, architecture, and design patterns. The repository aims to deploy an Apache Spark cluster with HDFS within a Kubernetes environment, enabling the execution of big data workloads using the Intel HiBench benchmark suite. The cluster functions as a regular Spark and HDFS cluster when not running benchmarks.

**Key Functionalities and Components:**

The system provides distributed data processing (Apache Spark) and storage (HDFS) capabilities, orchestrated by Kubernetes. Key components include:

*   **Apache Spark Cluster:** For distributed data processing.
*   **HDFS (Hadoop Distributed File System):** For distributed storage.
*   **Kubernetes Deployment:** Manages the deployment, scaling, and networking of the Spark and HDFS components using `kubernetes/cluster.yml` to define namespaces, services, pods, and StatefulSets.
*   **Intel HiBench:** A benchmark suite for evaluating big data performance. It supports Micro, Machine Learning, and Websearch workloads.
*   **Automated Configuration:** Python scripts and environment variables automate the configuration of HiBench and Hadoop/Spark within the containers. No manual DataNode registration is required.
*   **Node Affinity:** Kubernetes node labels are used to assign specific nodes as "master" nodes for services like NameNode, ResourceManager, and HistoryServer.

**Important Files and Their Roles:**

Several files play crucial roles in the repository's functionality:

*   **`LICENSE`:** Defines the project's Apache 2.0 license.
*   **`README.md`:** Provides an overview, build/run instructions, and references.

*   **`hadoop/base/create_config.py`:** Generates Hadoop configuration XML files from environment variables (prefixed with `CORE_CONF_`).
*   **`hadoop/base/hadoop.env`:** Defines environment variables for Core, HDFS, Yarn, and MapReduce configurations.
*   **`hadoop/{namenode,datanode,resourcemanager,historyserver}/Dockerfile`:** Dockerfiles for specific Hadoop components, extending `hadoop/base`.

**Architectural Insights:**

The architecture leverages:

*   **Containerization:** Docker containers provide modularity and portability for each Hadoop and Spark component.
*   **Kubernetes Orchestration:** Kubernetes manages the deployment, scaling, and networking of the cluster using Services and StatefulSets.
*   **Configuration via Environment Variables:** Hadoop and HiBench configurations are managed using environment variables for flexibility.
*   **Code Reuse:** The `hadoop/base` image serves as a common foundation for all Hadoop components.
*   **Separation of Concerns:** Dockerfiles build images, while shell scripts handle runtime configuration and daemon startup.
*   **Co-location of HiBench, NameNode, and Spark Master:** These components are placed within the same container to optimize performance based on locality of reference and to simplify HiBench integration with Hadoop and Spark directories.

**Notable Design Patterns:**

The repository employs several design patterns:

*   **Infrastructure as Code (IaC):**  `kubernetes/cluster.yml` defines the infrastructure declaratively, enabling consistent deployments.
*   **Configuration as Code:** Environment variables and Python scripts programmatically manage configuration.
*   **Microservices (to some degree):** The separation of Hadoop components into individual containers promotes modularity.
*   **Sidecar:** The DataNode pod contains both the Hadoop DataNode container and a YARN NodeManager container.
