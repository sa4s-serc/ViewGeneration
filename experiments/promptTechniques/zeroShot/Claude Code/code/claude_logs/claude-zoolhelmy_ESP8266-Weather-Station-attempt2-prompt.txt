ARCHITECTURAL SUMMARY:
## Weather Station Project: Analysis Report

This report summarizes the architecture, key functionalities, and components of an ESP8266-based weather station project. The system collects environmental data from various sensors, publishes it to both a cloud IoT platform (ThingSpeak) and a local OpenWRT router, and automatically generates and publishes historical data graphs to a GitHub repository. It also provides a local HTTP interface for real-time readings.

**Core Functionalities:**

The weather station's primary functions include:

*   **Sensor Data Acquisition:** Gathering data such as temperature, humidity, pressure, altitude, UV index, and potentially gas concentration and noise levels (hardware limitations might impact the latter two).
*   **Data Publication:** Transmitting sensor readings to ThingSpeak via MQTT and to the OpenWRT router, also via MQTT.
*   **Local HTTP Server:** Providing a web-based interface for accessing real-time sensor readings within the local network.
*   **Data Logging & Visualization:** Leveraging OpenWRT, collectd, and RRDTool to store and visualize sensor data locally.
*   **Automated Graph Generation and Publication:** Utilizing a daily cron job to generate historical graphs from RRDTool data and push them to a designated GitHub repository.

**Key Components and Files:**

The project's functionality is distributed across several key files and components:

*   **`README.md`:** Serves as the central project documentation, outlining hardware setup, software configuration, architectural overview, and relevant resources.
*   **`firmware/ESP8266-Weather-Station.ino`:** The Arduino sketch running on the ESP8266 NodeMCU. It encapsulates the core logic for sensor data acquisition, MQTT publishing, and the local HTTP server.
*   **`scripts/etc/init.d/mosquitto-subscribe`:** An OpenWRT init script responsible for starting the `mosquitto-subscribe.sh` script as a background service upon system boot.
*   **`scripts/etc/collectd/mosquitto-subscribe.sh`:** A shell script executed on the OpenWRT router that subscribes to MQTT topics published by the ESP8266. It then saves received sensor data to temporary files for processing by collectd.
*   **`scripts/etc/collectd/collectd-rrdtool-publish.sh`:**  A shell script executed by collectd on OpenWRT.  It reads the data from temporary files, updates the RRDTool database, and subsequently removes the files.
*   **`scripts/etc/collectd/rrdtool-graph-github.sh`:** A crucial shell script executed daily as a cron job on the OpenWRT router. Its purpose is to generate graphs from the RRDTool database, commit the changes to a Git repository, and push the updated graphs to the remote GitHub repository.
*   **`scripts/www/luci-static/resources/statistics/rrdtool/definitions/exec.js`:** A collectd configuration file that defines how the `luci-statistics` web interface on OpenWRT should generate graphs. It specifies the data to be graphed and the desired formatting.
*   **`data/*.txt`:** Temporary text files on the OpenWRT router used for transferring data between the mosquitto subscriber and collectd for further processing by rrdtool.

**Architectural Overview:**

The system employs a hybrid architecture, leveraging both cloud (ThingSpeak) and local (OpenWRT) infrastructure. This approach offers redundancy and enables both real-time and historical data analysis. MQTT serves as the central communication protocol, facilitating the transmission of sensor data from the ESP8266 to both ThingSpeak and the OpenWRT router. The OpenWRT router functions as a local processing hub, handling data logging, visualization, and automated graph generation. This reduces dependence on cloud services and enables localized data processing. Automation is achieved through cron jobs on the OpenWRT router, scheduling tasks like graph generation and GitHub publishing.

**Design Patterns:**

The project demonstrates several notable design patterns:

*   **Publisher-Subscriber:** The ESP8266 acts as a publisher, broadcasting sensor data to MQTT topics. ThingSpeak and the OpenWRT router act as subscribers, receiving and processing this data.
*   **Data Buffering:** The temporary storage of data in text files on the OpenWRT router facilitates the transfer of data between processes (mosquitto subscriber and collectd). This represents a simple form of data buffering.
*   **Scheduled Task:** The `rrdtool-graph-github.sh` script employs a scheduled task (cron job) to regularly generate and publish graphs. This ensures consistent and automated data visualization and reporting.
