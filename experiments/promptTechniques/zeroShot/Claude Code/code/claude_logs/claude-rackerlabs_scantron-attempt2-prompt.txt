**Scantron Project: Consolidated Analysis Report**

This report consolidates the analysis of the Scantron project, a distributed network scanner designed for scheduling, distributing, executing, and processing network scans. The architecture comprises a console (web front-end) and engines (scanning nodes) communicating via a REST API, with Ansible playbooks used for automated deployments.

**Core Functionality & Components:**

The system's core lies in **scan scheduling**, driven by `django-recurrence` for complex recurrence rules. The `scan_scheduler.py` script is central to this, managing scan instances based on database configurations and scheduling logic.  **Distributed scanning** distributes tasks to engines via a REST API, where engines poll the console for new jobs.

Engines then **execute scans** using `nmap` or `masscan`.  **Result handling** involves engines storing scan results, often on a shared NFS volume, which the console processes. Scripts within the `scan_results` directory (`nmap_to_csv.py`, `masscan_json_to_csv.py`, `merge_nmap_xml_files.py`, `merge_masscan_json_files.py`) parse scan results (XML/JSON) into CSV format for use with big data analytics platforms. This integration is facilitated by storing the resulting CSV files within the `/console/for_bigdata_analytics/` directory. `extract_targets.py` provides functions for target extraction and validation.

The system employs a **REST API** (built with Django REST Framework) for communication between the console and engines, enabling external systems to schedule and monitor scans. API tokens are used for authentication. An API client (`scantron_api_client.py`) allows programmatic interaction.  The system also supports **email alerts** upon scan completion, errors, or scan diffs.

**Key Files & Roles:**

*   **Project Foundation:** `LICENSE` (Apache 2.0 license) defines terms of use. `README.md` provides a project overview, deployment instructions, and API documentation. `initial_setup.sh` handles initial development environment setup.

*   **Engine Components:** The `engine/` directory houses the scanning engine. `engine.py` is the main engine script. `scantron-engine.service` facilitates running the engine as a service. `nfs_watcher.sh` ensures the NFS share is mounted.

*   **Ansible Automation:** `ansible-playbooks/` contains playbooks for console (`console.yml`) and engine (`engine.yml`) deployment, ensuring repeatable configurations. `roles/` directories (common, add_users, console, engine) define modular automation tasks.

*   **Console (Django Application):** The `console/` directory contains the Django project.  `django_scantron/models.py` defines Django models (Configuration, Engine, Site, Scan, ScheduledScan). `scan_retention.py` handles data cleanup based on retention policies. `utility.py` offers utility functions. `scan_scheduler_visualizer.py` visualizes scan schedules.

*   **Frontend Enhancements:** `jquery.dataTables.js` enhances HTML tables with features like data retrieval, sorting, and pagination. `font-awesome.min.css` provides icon styles. `jquery-ui.min.js` offers UI widgets.

*   **Target Files:** `/console/target_files/README` indicates a directory for storing IP addresses for sites to scan.
