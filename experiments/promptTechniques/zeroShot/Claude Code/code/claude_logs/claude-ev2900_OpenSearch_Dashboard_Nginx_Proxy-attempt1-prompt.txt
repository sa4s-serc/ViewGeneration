## Report: ev2900_OpenSearch_Dashboard_Nginx_Proxy Repository Analysis

This repository provides a solution for securely accessing an OpenSearch dashboard deployed in a private subnet via the public internet using an Nginx proxy server. **Functionality:**

The core functionality is enabling secure public access to OpenSearch dashboards hosted within a private subnet. This is achieved by using Nginx as a reverse proxy to forward incoming HTTPS traffic to the OpenSearch dashboard, effectively hiding its internal location and providing a single point of entry. SSL/TLS is employed, utilizing self-signed certificates for encryption between the user and the Nginx proxy server.

**Architecture:**

The architecture utilizes a tiered approach: A private OpenSearch domain, an Nginx proxy server in a public subnet within the same VPC, and controlled traffic flow via Security Groups. **Design Patterns:**

*   **Reverse Proxy:**  Nginx shields the internal infrastructure by acting as a reverse proxy.
*   **Infrastructure as Code (IaC):** The CloudFormation template enables repeatable and automated deployments, facilitating infrastructure management.
