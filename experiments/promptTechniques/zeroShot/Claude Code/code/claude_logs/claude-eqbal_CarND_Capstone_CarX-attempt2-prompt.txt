## Self-Driving Car ROS Project Report

This report summarizes the key aspects of a self-driving car project implemented using the Robot Operating System (ROS). The project encompasses perception, planning, and control functionalities, designed for both simulation (Udacity simulator) and real-world operation (CARLA).

**Key Functionalities and Components:**

The project is structured around several key ROS nodes, each responsible for a specific task:

*   **Perception:**
    *   **Traffic Light Detection Node:** Detects and classifies traffic light states from camera images using a `tl_classifier.py` script. This classifier identifies traffic light color based on HSV color space. Camera calibration is handled via `camera_info_publisher` using data in `grasshopper_calibration.yml`.
*   **Planning:**
    *   **Waypoint Loader:** Loads waypoint data from a CSV file (`wp_yaw.txt`) and publishes it as a ROS topic, which are loaded with `waypoint_loader.py`.
    *   **Waypoint Updater:** Selects and adjusts the velocity of waypoints based on traffic light status and obstacles, implemented in `waypoint_updater.py`.
    *   **Waypoint Follower:** Implements the pure pursuit algorithm (C++ implementation in `libwaypoint_follower.cpp`, `pure_pursuit.cpp`, `pure_pursuit_core.cpp`) to follow waypoints.
*   **Control:**
    *   **Twist Controller (DBW):** Controls vehicle actuators (steering, throttle, brake) based on target velocities. This is implemented within `dbw_node.py`.
    *   **PID Controller:** A `pid.py` script contains the implementation for a PID controller used for throttle control.
    *   **Yaw Controller:** Implemented in `yaw_controller.py` and is used for steering control.
    *   **LowPass Filter:** Implemented in `lowpass.py`.
*   **Infrastructure:**
    *   **Bridge Node:** Connects the ROS environment to the Unity simulator. Implemented using Flask and SocketIO in `server.py` with configuration defined in `conf.py` and bridge functionality in `bridge.py`.

**Important Files and Their Roles:**

*   **`README.md`:** Provides a comprehensive project overview, architecture description, ROS node details, installation and usage instructions, collaboration guidelines, team information, and testing procedures.
*   **`requirements.txt`:** Specifies Python dependencies, including Flask, numpy, scipy, keras, and tensorflow.
*   **`ros/visualizer.rviz`:** RVIZ configuration file used for visualizing the self-driving car's perception, planning, and control data for debugging.
*   **`ros/launch/styx.launch`:** ROS launch file for running the project with the Udacity simulator, including the simulator bridge and relevant nodes.
*   **`ros/launch/site.launch`:** ROS launch file for running the project on the Udacity's real car (CARLA), including the DBW node and relevant nodes.
*   **`ros/src/styx/CMakeLists.txt`:** CMake build file for the `styx` ROS package.

**Architectural Insights:**

The project adopts a modular architecture built on ROS. Each ROS node represents a distinct functionality, enabling independent development and testing. Communication between nodes occurs through ROS topics and messages, promoting separation of concerns. The inclusion of separate launch files (`styx.launch` and `site.launch`) for the simulator and the real car highlights the project's adaptability to different operational environments.

**Notable Design Patterns:**

*   **Publisher-Subscriber:** ROS nodes communicate via publishing messages to topics and subscribing to relevant topics.
*   **Model-View-Controller (MVC)-like:** A separation of concerns mirrors the MVC pattern, where the simulator functions as the model, ROS nodes provide control logic, and RVIZ offers visualization.
*   **PID Control:** The twist controller uses a PID controller for throttle regulation, based on velocity error.
*   **Pure Pursuit:** The waypoint follower employs the pure pursuit algorithm for path tracking.
