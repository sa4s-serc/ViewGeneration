## Analysis of GitHub Repository: marekq_rss-lambda

This report analyzes the `marekq_rss-lambda` GitHub repository, which defines an AWS Serverless application designed to monitor RSS feeds, extract content from blog posts, and provide notifications and data access. It utilizes a combination of Lambda functions, DynamoDB, S3, Step Functions, and optionally, AppSync and Algolia.

**Key Functionalities and Components:**

*   **RSS Feed Monitoring:** Periodically retrieves and parses RSS feeds from a predefined list of sources (configured in `feeds.txt`).
*   **Content Extraction:** Extracts the main content of blog posts using the `readability-lxml` library.
*   **Data Storage:** Stores extracted blog post data (title, content, metadata) in a DynamoDB table.
*   **Notifications:** Sends email notifications via SES when new blog posts are found (optional, controlled by `SendEmails` parameter in `template.yaml`).
*   **Data Access:** Provides read-only access to blog post data via a GraphQL API powered by AppSync (optional, enabled with `CreateAppSync` parameter). Resolvers for the AppSync API are defined in the `graphql/*` directory, using VTL.
*   **Search Indexing:** Optionally indexes blog posts using Algolia for search functionality (controlled by `EnableAlgolia`, `AlgoliaApp`, `AlgoliaApikey`, `AlgoliaIndex` parameters).
*   **Public S3 Storage:** Optionally stores a JSON object of blog posts as a public S3 file (`StorePublicS3` parameter).
*   **Entity Recognition:** Uses AWS Comprehend to perform entity recognition on the extracted blog post content, generating tags.
*   **Article Count Tracking:** The `lambda-pagecount/*` function retrieves and stores the total article count for each blog in DynamoDB. This function is invoked manually.

**Important Files and Their Roles:**

*   **`template.yaml`:** The SAM (Serverless Application Model) template that defines the AWS infrastructure. *   **State Machine Orchestration:** A Step Function coordinates the overall workflow of retrieving feeds and processing blog posts.

**Notable Design Patterns:**

*   **Data Lake/Data Warehouse:** DynamoDB acts as a data store for the extracted blog post information. S3 acts as a data lake for public consumption of JSON-based blog data.
*   **Lambda Layers:** Uses Lambda Layers to manage dependencies, improving deployment size and speed.
