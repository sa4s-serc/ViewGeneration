## UnoCash Application Analysis Report

This report provides an overview of the UnoCash application, a personal finance tool built using Blazor for the client-side and Azure Functions for the serverless backend. It consolidates features such as transaction storage, shared expense tracking, receipt recognition, and potential mobile payment integration, leveraging Azure services for a modern, cloud-native architecture. The application uses Pulumi for Infrastructure as Code.

**1. Overview:**

UnoCash aims to simplify personal finance management by providing a unified platform for various tasks. It leverages a serverless microservices architecture with a Blazor-based SPA frontend for a rich and interactive user experience.

**2. Key Functionalities and Components:**

*   **Transaction Management:** Stores and retrieves financial transactions (expenses).
*   **Receipt Parsing:** Extracts data from receipt images using Azure Form Recognizer. Caching is implemented to avoid re-processing receipts.
*   **Expense Tracking:** Enables tracking of shared expenses.
*   **Reporting and Visualization:** Provides reports and graphs (implementation details not present in snippets).
*   **Authentication and Authorization:** Uses Azure AD for authentication and planned integration with Azure AD B2C for identity management.
*   **Deployment:** Uses Pulumi for infrastructure as code, automating the provisioning and configuration of cloud resources.

**2.1 Frontend (UnoCash.Blazor):**

The Blazor frontend provides the user interface for managing expenses. It includes:

*   **Pages:** Index (Home), FetchData (Expense List), Update (Expense Add/Edit).  Pages are defined as `.razor` files. The `Update.razor.cs` file handles data binding, API calls, and receipt uploading logic.
*   **Shared Components:** MainLayout, NavMenu, defining the application's layout and navigation structure.
*   Uses a set of Bootstrap components from `blazorstrap`.
*   `wwwroot/css/site.css`: Main CSS file, utilizing open-iconic icons and Bootstrap.
*   `azure-storage.blob.min.js`: JavaScript library for Azure Blob Storage operations, enabling receipt uploading.

**2.2 Backend (UnoCash.Api):**

The Azure Functions backend handles API requests, interacts with data storage, and performs receipt parsing. It includes functions for:

*   GetExpenses, AddExpense, UpdateExpense, DeleteExpense, GetReceiptData, GetReceiptUploadSasToken. These are triggered by HTTP requests.
*   The ReceiptParser uses the Form Recognizer (mentioned but not directly integrated in the snippet) to extract information from uploaded images.

**2.3 Infrastructure as Code (UnoCash.Pulumi):**

The Pulumi project (written in F#) defines and provisions the cloud infrastructure:

*   Azure Resource Group, Storage Account, Function App Service Plan, API Management, Azure AD Application, and DNS records.

*   Leverages helper functions/modules defined in `Async.fs`, `Output.fs`, and `Certificate.fs`.

**3. Core Components (UnoCash.Core):**

The core project contains the main application logic:

*   `ExpenseReader.cs`, `ExpenseWriter.cs`: Manages reading and writing expense data to Azure Table Storage (abstracted as repository pattern).
*   `ReceiptParser.cs`: Parses receipt images using Azure Form Recognizer and stores results in Azure Table Storage.
