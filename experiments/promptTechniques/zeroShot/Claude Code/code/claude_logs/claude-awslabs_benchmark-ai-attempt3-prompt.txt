**Benchmark AI (BAI) System Overview**

The `awslabs_benchmark-ai` repository outlines a system for defining, executing, monitoring, and reporting on AI/ML benchmark jobs, primarily within a Kubernetes environment on AWS. It adopts a microservices architecture, leveraging Kafka for asynchronous communication between components.  Key functionalities include job submission, data fetching, metrics collection, and infrastructure management.

**I. Core Components and Functionalities**

*   **`bai-bff` (Backend For Frontend):**  Handles benchmark job submissions, validates descriptors (TOML), manages job lifecycle (submission, status, results, cancellation), and exposes HTTP API endpoints. Utilizes Clojure (Compojure, Stuart Sierra Component model). Manages event bus channels using core.async.
*   **`executor`:** Orchestrates benchmark job execution on Kubernetes. Translates benchmark descriptors into Kubernetes resources (Jobs, CronJobs, Services, ConfigMaps, MPIJobs).  Handles distributed training strategies (single-node, Horovod, inference).
*   **`watcher`:** Monitors the execution of benchmark jobs on both Kubernetes and SageMaker.  Infers job status using strategies like `SingleNodeStrategyKubernetesStatusInferrer`.  Publishes status updates to Kafka.
*   **`sm-executor`:** Executes benchmark jobs on AWS SageMaker, managing the lifecycle of SageMaker training jobs.
*   **`fetcher` and `fetcher-dispatcher`:** Downloads datasets and models from various sources (HTTP, S3) to persistent volumes, managed by `fetcher-dispatcher` which creates fetcher jobs on Kubernetes. Uses ZooKeeper for service discovery and coordination during data fetching.
*   **`cloudwatch-exporter`:** Collects and exports user-defined metrics to AWS CloudWatch.
*   **Metrics Components:**
    *   `metrics-pusher`: A sidecar container that collects metrics from benchmark jobs and pushes them to Kafka.
    *   `metrics-extractor`: Extracts metrics from pod logs using a dedicated microservice.
    *   `client-lib`: A Python API for emitting metrics from within benchmark code.

**II. Infrastructure and Deployment**

*   **Kubernetes Deployments:** Kustomize manages Kubernetes deployments, customizing configurations for different environments. YAML files define deployments for Zookeeper, Prometheus, etc.
